Confirm

This function confirms a user sign up via a unique confirmation token.

When a new user signed up, a confirmation email will be sent to the registered email address.
(Make sure Autoconfirm isn't turned on under identity settings.)

In the email, there's a link that says Confirm your email address. When a user clicks on the link, it'll be redirected to the site of the site with a fragment identifier that looks like #confirmation_token=Iyo9xHvsGVbW-9A9v4sDmQ.

For security reason, the confirmation_token is hidden from the browser via a redirect. If you wish to manually confirm a user using the auth.confirm(token) method, you can copy the link location of the email and use the curl script to get the confirmation token in your terminal. E.g., 

                    
$ curl -I https://mandrillapp.com/track/click/30002868/example.netlify.com?p=example-token
HTTP/1.1 302 Moved Temporarily
Server: nginx/1.12.2
Date: Tue, 15 May 2018 21:19:13 GMT
Content-Type: text/html; charset=utf-8
Set-Cookie: PHPSESSID=77c421bf85fa412e5f994f28a6b30956; expires=Wed, 16-May-2018 07:19:13 GMT; path=/; secure; HttpOnly
Expires: Thu, 19 Nov 1981 08:52:00 GMT
Cache-Control: no-store, no-cache, must-revalidate, post-check=0, pre-check=0
Pragma: no-cache
Set-Cookie: PHPSESSID=77c421bf85fa412e5f994f28a6b30956; expires=Wed, 16-May-2018 07:19:13 GMT; path=/; secure; httponly
Location: https://example.netlify.com/#confirmation_token=Iyo9xHvsGVbW-9A9v4sDmQ
